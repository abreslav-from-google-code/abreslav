Автоматическое извлечение языков запросов для поиска по моделям.

Введение
  * Модель и метамодель, структура модели
    * Какая у нас метамодель, что такое модель - дерево с типизир. узлами и именованными ребрами с множественностью
  * Задача поиска, мотивация
    * Нужно искать в модели части, удовлетворяющие шаблону
      * а как вообще иначе искать :) - только по тексту полей
    * Зачем? Для навигации, для преобразований (XSLT), ЕЩЕ?!
  * Операторы для поиска 
    * Что такое "шаблон": части модели с "дырками" - какие бывают дырки: просто так и переменные
  * Язык запросов
    * Итого: язык позволяет задавать шаблоны и искать по ним

(Абстрактный) Синтаксис языка запросов
  * Двойственность метамодели
    * Основные типы совпадают, появляются расширения - для переменных и подстановочных знаков
  * Подстановочные символы и их свойства
    * ограниченная жадность
  * Коммутативные операции
    * порядок сравнения, ровно одна подстановка - то, есть просто флаг

Переменные
  * Типизация
    * Определяется сопоставленным шаблоном
  * Область действия
    * ???

Семантика запросов
  * Прямое соответствие
    * тупо сравниваем тип вершины и значения атрибутов на равно
  * Подстановочные символы для некоммутативных операций
    * ищем цепочку до первого стоп-совпадения, если есть следующий и до конца, если нет
  * -"- для коммутативных операций
    * все плохо...
  * Переменные
    * один раз сопоставили - и дальше на структурное совпадение

Автоматическое построение языка запросов
  * Синтаксис
